# รายงานสถานะการรับทราบหลักสูตร

Static dashboard สำหรับเผยแพร่บน GitHub Pages โดยอ่านข้อมูลจาก Google Sheet ผ่าน Google Apps Script endpoint

## สิ่งที่ปรับในเวอร์ชันนี้

- เปลี่ยนชื่อ Dashboard เป็น `รายงานสถานะการรับทราบหลักสูตร`
- เปลี่ยนข้อความสถานะสำเร็จเป็น `หลักสูตรได้รับการพิจารณารับทราบจาก สป.อว. แล้ว`
- เปลี่ยนข้อความรายการย่อยเป็น `ปรับปรุงเล็กน้อย สมอ.08 กรณีเปลี่ยนแปลงอาจารย์`
- กรณีข้อมูลมี `ครั้งที่ ...` ระบบจะต่อท้ายข้อความเดิมอัตโนมัติ
- เปลี่ยนชื่อ KPI เป็น `อัตราการรับทราบหลักสูตร มคอ.2 จาก สป.อว.`
- รองรับและแสดงคอลัมน์ `นำส่งผ่านเอกสาร/ ระบบ`
- เพิ่มตัวกรองช่องทางการนำส่ง และแสดงช่องทางในตารางรายละเอียด ตารางรายการที่อยู่ระหว่างพิจารณา และไฟล์ Export CSV
- ปรับประวัติรายการ สมอ.08 ให้แสดงแต่ละครั้งแยกคนละบรรทัด โดยไม่ใช้เครื่องหมาย `/` และแสดงวันที่กำกับแต่ละรายการ

## วิธีอัปโหลดขึ้น GitHub ใหม่

1. สร้าง Repository ใหม่ใน GitHub
2. แตกไฟล์ ZIP แล้วอัปโหลดไฟล์ทั้งหมด โดยให้ `index.html` อยู่ที่ root ของ Repository
3. Commit ไปที่ branch `main`
4. ไปที่ `Settings > Pages`
5. ในหัวข้อ `Build and deployment` เลือก Source เป็น `GitHub Actions`
6. รอ Workflow `Deploy static content to Pages` ทำงานเสร็จ
7. เปิด URL ที่ GitHub Pages สร้างให้

ไฟล์ `.github/workflows/static.yml` ได้เตรียมไว้แล้วสำหรับ Deploy อัตโนมัติเมื่อมีการ Push เข้า branch `main`

## รูปแบบ JSON ที่รองรับ

```json
{
  "status": "success",
  "count": 298,
  "data": [
    {
      "หน่วยงาน": "...",
      "คณะ": "...",
      "วิทยาเขต": "...",
      "รหัสติดตาม": "...",
      "รหัสหลักสูตร": "...",
      "ชื่อหลักสูตร": "...",
      "ระดับการศึกษา": "...",
      "วันที่รับทราบ": "...",
      "ประเภทการดำเนินการ": "...",
      "เกณฑ์ประเมิน": "...",
      "นำส่งผ่านเอกสาร/ ระบบ": "CHECO"
    }
  ]
}
```

ค่าที่รองรับในคอลัมน์ `นำส่งผ่านเอกสาร/ ระบบ` ได้แก่

- `ส่งเอกสารต้นฉบับ`
- `CISA`
- `CHECO`

Dashboard ยังรองรับ Response ที่เป็น Array โดยตรง หรือใช้ key `rows` แทน `data`

## Logic หลัก

- จัดกลุ่มข้อมูลตาม `เกณฑ์ประเมิน`
- ประเภทการดำเนินการที่นับเป็น มคอ.2 ได้แก่ `หลักสูตรใหม่`, `ปรับปรุงตามกำหนดรอบปรับปรุง`, `ปรับปรุงแบบแยก`, `ปรับปรุงแบบรวม`
- ถ้า `วันที่รับทราบ` มีวันที่ ระบบจัดเป็น `หลักสูตรได้รับการพิจารณารับทราบจาก สป.อว. แล้ว`
- ถ้า `วันที่รับทราบ` เป็น `อยู่ระหว่างการพิจารณา` หรือว่าง ระบบจัดเป็นรายการที่อยู่ระหว่างการพิจารณา
- รายการ `ปรับปรุงเล็กน้อย สมอ.08 กรณีเปลี่ยนแปลงอาจารย์ ครั้งที่ ...` นับเป็นจำนวนครั้ง และไม่นำไปรวมกับหลักสูตรที่ได้รับการพิจารณารับทราบจาก สป.อว. แล้ว

- ประวัติการปรับปรุงเล็กน้อยแสดงเป็นกล่องต่อกัน โดยแต่ละกล่องมีเลขครั้งและวันที่
